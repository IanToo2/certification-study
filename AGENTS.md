# Repository Guidelines

## 프로젝트 목적
이 저장소는 AWS Solutions Architect Associate(SAA-C03) 시험 대비용 한국어 스터디북을 생성하는 프로젝트다. 모든 문서는 Markdown으로 작성하며, 최종 결과물은 PDF로 변환된다는 점을 전제로 구성한다.

## 언어 및 표현 규칙
설명 문장은 반드시 한국어로 작성한다. 다만 AWS 서비스명은 `Amazon EC2`, `AWS Lambda`, `Amazon RDS`처럼 English 표기를 유지한다. 문체는 학습서 기준으로 간결하고 설명적이어야 하며, 시험 준비에 직접 도움이 되는 내용만 남긴다.

## 서비스별 필수 구성
각 AWS 서비스 문서에는 아래 8개 항목이 반드시 포함되어야 한다.

1. 서비스 개념
2. 아키텍처 구조
3. 언제 사용하는가
4. 언제 사용하면 안되는가
5. 비슷한 서비스와 차이
6. 시험에서 자주 나오는 문제 유형
7. 시험 함정 포인트
8. 실무 아키텍처 예시

누락된 항목이 있으면 초안으로 간주한다. 비교 항목에서는 대안 서비스와의 선택 기준을 분명히 적는다.

## 아키텍처 작성 규칙
가능하면 ASCII 다이어그램 또는 Mermaid 다이어그램을 사용한다. 다이어그램은 단순한 그림이 아니라 트래픽 흐름, 고가용성 구성, 계층 간 역할이 드러나야 한다.

예시:

```text
Client
  |
  v
CloudFront
  |
  v
ALB
  |
  v
EC2
  |
  v
RDS
```

## 시험 대비 중심 작성 원칙
모든 장과 서비스 설명은 시험 준비에 초점을 맞춘다. 특히 아래 4가지를 반복적으로 강조한다.

- 아키텍처 의사결정 근거
- 서비스 간 비교와 선택 기준
- 자주 나오는 함정 포인트
- 고가용성 및 비용 최적화 패턴

실무 설명도 중요하지만, 항상 “시험에서 어떻게 출제되는가”까지 연결해서 마무리한다.

## 파일 및 콘텐츠 운영
콘텐츠는 Markdown 파일로 관리한다. 새 문서를 추가할 때는 서비스 단위 또는 주제 단위로 분리하고, 제목과 섹션 이름은 일관되게 유지한다. PDF 변환을 고려해 표, 코드 블록, 다이어그램은 렌더링이 깨지지 않는 형식을 우선 사용한다.
